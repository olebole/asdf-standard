This reference section describes the schema files for the built-in tags in ASDF.

ASDF schemas are arranged into “modules”. All ASDF implementations must support the “core” module, but the other modules are optional.

  • Core
    • asdf: Top-level schema for every ASDF file.
    • complex: Complex number value.
    • ndarray: An n-dimensional array.
    • table: A table.
    • column: A column in a table.
    • constant: Specify that a value is a constant.
    • software: Describes a software package.
    • history_entry: An entry in the file history.
  • FITS
    • fits: A FITS file inside of an ASDF file.
  • Unit
    • unit: Physical unit.
    • defunit: Define a new physical unit.
  • Time
    • time: Represents an instance in time.
  • Transform
    • Basics
    • Compound transformations
    • Arithmetic operations
    • Simple Transforms
    • Projections
    • Polynomials
    • Regions and labels
  • WCS
    • wcs: A system for describing generalized world coordinate transformations.
    • step: Describes a single step of a WCS transform pipeline.
    • frame: The base class of all coordinate frames.
    • celestial_frame: Represents a celestial frame.
    • spectral_frame: Represents a spectral frame.
    • composite_frame: Represents a set of frames.
